üîÄ GitHub Actions Flow

  1. PR opened/updated on GitHub
  2. Actions workflow triggers (visual_tests.yml)
  3. Detect staging URL from PR labels or secrets
  4. Wait for staging to be accessible
  5. Download approved baselines from artifacts cache
  6. Run run_visual_ci.py for each configured page
  7. Compare screenshots using SSIM + CV2
  8. Post ‚úÖ/‚ùå commit status check
  9. Post detailed PR comment with table + links
  10. Upload artifacts (diffs, PDFs, JSON summary)

‚ö° Server Webhook Flow

  1. PR event webhook received at /api/github/webhook
  2. HMAC-SHA256 signature verified
  3. Staging URL resolved from PR label or config
  4. Pending status posted to commit immediately
  5. Background thread spawned per PR
  6. Screenshots taken via Playwright
  7. Images compared via SSIM + semantic classification
  8. PDF + diff overlay generated per page
  9. PR comment upserted (update existing bot comment)
  10. Final ‚úÖ/‚ùå commit status posted

üè∑ Staging URL Detection (Priority Order)

  1. PR Label ‚Äî Add label staging-url:https://pr-123.staging.example.com to any PR
  2. STAGING_BASE_URL secret ‚Äî Set in GitHub Secrets + /pr-{number} appended
  3. STAGING_URL secret ‚Äî Fixed staging URL used for all PRs
  4. Manual trigger ‚Äî Enter URL in the CI Dashboard or workflow_dispatch input
